Activity: Data Modeling Challenge with MongoDB Atlas

Objective

Design a database schema for an online learning platform using MongoDB Atlas, populate collections with data, write and execute queries to retrieve information, and submit your work by exporting the database and pushing the results to GitHub.

Scenario: Online Learning Platform

Scenario Details

The platform requires the following:

  • Courses:
    • Each course has a title, description, and a list of enrolled students.
    • Courses are organized into categories: Programming, Data Science, and Design.
  • Students:
    • Students have a name, email, and the list of courses they are enrolled in.
    • Students can leave reviews for courses.
  • Instructors:
    • Each course is taught by one instructor.
    • Instructors have a name, email, and the list of courses they teach.
  • Reviews:
    • Students can rate courses (1 to 5 stars) and leave comments.
    • Reviews are tied to a specific course and a specific student.

Instructions

Step 1: Prepare Your MongoDB Database

  1. Open MongoDB Atlas and navigate to your existing cluster.
  2. Access the Collections tab.
  3. Create a new database named online_learning.

Step 2: Populate the Database

Categories Collection
  • Collection Name: categories
  • Insert the Following Documents: 
    [
  { "name": "Programming" },
  { "name": "Data Science" },
  { "name": "Design" }
]
Instructors Collection
  • Collection Name: instructors
  • Insert the Following Documents: 
    [
  {
    "name": "John Smith",
    "email": "john.smith@example.com",
    "teachingCourses": []
  },
  {
    "name": "Jane Doe",
    "email": "jane.doe@example.com",
    "teachingCourses": []
  }
]
Students Collection
  • Collection Name: students
  • Insert the Following Documents: 
    [
  {
    "name": "Alice Johnson",
    "email": "alice.j@example.com",
    "enrolledCourses": []
  },
  {
    "name": "Bob Brown",
    "email": "bob.b@example.com",
    "enrolledCourses": []
  },
  {
    "name": "Charlie White",
    "email": "charlie.w@example.com",
    "enrolledCourses": []
  }
]
Courses Collection
  • Collection Name: courses
  • Insert the Following Documents: 
    [
  {
    "title": "Introduction to MongoDB",
    "description": "Learn the basics of MongoDB.",
    "category": ObjectId("CATEGORY_ID_1"),
    "instructor": ObjectId("INSTRUCTOR_ID_1"),
    "students": [ ObjectId("STUDENT_ID_1"), ObjectId("STUDENT_ID_2") ],
    "reviews": [
      { "rating": 5, "comment": "Great course!", "student": ObjectId("STUDENT_ID_1") },
      { "rating": 4, "comment": "Very informative.", "student": ObjectId("STUDENT_ID_2") }
    ]
  },
  {
    "title": "Data Science with Python",
    "description": "A comprehensive guide to data science using Python.",
    "category": ObjectId("CATEGORY_ID_2"),
    "instructor": ObjectId("INSTRUCTOR_ID_2"),
    "students": [ ObjectId("STUDENT_ID_3") ],
    "reviews": [
      { "rating": 4, "comment": "Good introduction.", "student": ObjectId("STUDENT_ID_3") }
    ]
  }
]
    Note: Replace CATEGORY_ID_1, INSTRUCTOR_ID_1, and other ObjectId placeholders with actual _id values from the respective collections.

Step 3: Write and Execute Queries

  1. Retrieve all courses in the "Programming" category.
  2. Retrieve all students enrolled in the course "Introduction to MongoDB".
  3. Retrieve course details, including category and instructor information, for all courses.
  4. Retrieve all reviews for the course "Data Science with Python," including the details of the students who wrote them.
  5. Find the average rating for each course.
  6. Retrieve all courses taught by the instructor "Jane Doe".
  7. Count the number of courses in each category.
  8. Find all students who have reviewed the course "Introduction to MongoDB".
  9. Retrieve the titles of all courses where the average rating is above 4.
  10. Retrieve all students who are enrolled in courses taught by "John Smith".

Step 4: Export the Database

  1. Open your MongoDB Atlas cluster and navigate to the Collections tab.
  2. Click on the Export option for the online_learning database.
  3. Export all collections as JSON files.
  4. Ensure that your exported files include all populated collections: categories, instructors, students, and courses.

Step 5: Submit Your Work

  1. Create a folder in your GitHub repository under the path: Day11 > Activity >.
  2. Name the folder online_learning_export.
  3. Include the following in the folder:
    • All exported JSON files of your database.
    • A text file named activity.txt containing the results of your 10 queries.
  4. Push the folder to your GitHub repository.

Grading Criteria

  • Schema Design: Proper use of embedding and referencing (30%).
  • Data Population: Accurate and complete sample data (30%).
  • Query Execution: Correct results for all queries (30%).
  • Export and Submission: Properly exported and submitted database with query results (10%).
